1.2.1.1.2.1. xsigmamodules.generate_pyi#

This program will generate .pyi files for all the XSIGMA modules in the “xsigmamodules” package (or whichever package you specify). These files are used for type checking and autocompletion in some Python IDEs.

The XSIGMA modules must be in Python’s path when you run this script. Options are as follows:

-p PACKAGE The package to generate .pyi files for [xsigmamodules] -o OUTPUT The output directory [default is the package directory] -e EXT The file suffix [.pyi] -i IMPORTER The static module importer (for static builds only) -b BINDIR The extras directory to create [default is none] -h HELP

With no arguments, the script runs with the defaults (the .pyi files are put inside the existing xsigmamodules package). This is equivalent to the following:

python -m xsigmamodules.generate_pyi -p xsigmamodules

To put the pyi files somewhere else, perhaps with a different suffix:

python -m xsigmamodules.generate_pyi -o /path/to/xsigmamodules -e .pyi

To generate pyi files for just one or two modules:

python -m xsigmamodules.generate_pyi -p xsigmamodules Core

To generate pyi files for your own modules in your own package:

python -m xsigmamodules.generate_pyi -p mypackage mymodule [mymodule2 ...]
