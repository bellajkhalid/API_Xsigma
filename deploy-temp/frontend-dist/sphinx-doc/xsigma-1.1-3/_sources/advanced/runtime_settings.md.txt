# Runtime settings

## OpenGL

On Linux and Windows, XSIGMA will attempt to detect support for an OpenGL context backend at runtime
and create an appropriate subclass of `xsigmaOpenGLRenderWindow`. You can override this process by
specifying an environment variable `XSIGMA_DEFAULT_OPENGL_WINDOW`. The possible values
are:

  1. `xsigmaXOpenGLRenderWindow` (Linux; applicable only when `XSIGMA_USE_X` is `ON`, which is the default setting)
  2. `xsigmaWin32OpenGLRenderWindow` (Windows; applicable only when `XSIGMA_USE_WIN32_OPENGL` is `ON`, which is the default setting)
  3. `xsigmaEGLRenderWindow` (applicable only when `XSIGMA_OPENGL_HAS_EGL` is `ON`, which is the default setting)
  4. `xsigmaOSOpenGLRenderWindow` (OSMesa, requires that `osmesa.dll` or `libOSMesa.so` is installed)

Note: XSIGMA does **not** support OSMesa on macOS, iOS, Android and WebAssembly platforms.
