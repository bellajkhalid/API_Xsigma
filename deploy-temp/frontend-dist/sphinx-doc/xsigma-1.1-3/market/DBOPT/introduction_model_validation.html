Introduction to Model and Validation#

🎯 Validation Scope#

Objectif de la Validation#

Cette section définit le scope complet de la validation pour les modèles DBOPT-RateDeterministic_InflationDeterministic et DBX-RateDeterministic_InflationDeterministic, incluant les produits couverts, les méthodologies de validation et les critères d’évaluation.

Modèles dans le Scope#

  • DBOPT-RateDeterministic_InflationDeterministic avec configuration DBOPT-RatesMarketDeterministic_InflationDeterministic

  • DBX-RateDeterministic_InflationDeterministic avec configuration DBX-ShortRateDeterministic_InflationDeterministic

Version du Modèle#

Le modèle a été revu pour DBAnalytics v1300 avec validation complète des fonctionnalités et performances.

📊 Product Description#

Caractéristiques Générales des Produits#

Propriétés Fondamentales#

Les produits couverts par ce framework partagent les caractéristiques suivantes :

  • Dépendance aux taux d’intérêt et à l’inflation

  • Absence d’optionalité sur les taux d’intérêt et l’indice d’inflation

  • Linéarité par rapport aux courbes de taux et d’inflation

  • Pricing déterministe basé sur des arguments de non-arbitrage

Structure Mathématique#

Les produits peuvent être représentés sous la forme générale :

\[\text{Payoff} = N \times f(\text{Interest Rate}, \text{Inflation Index})\]

où \(f\) est une fonction linéaire des facteurs de risque sous-jacents.

Produits Spécifiques#

🔄 Inflation Zero Coupon Swaps (InflSwapZC)#

Description : Swaps d’inflation zero-coupon échangeant un taux fixe contre l’inflation cumulée.

Formule de Payoff : $\(\text{Payoff} = N \times \left[\left(\frac{I_T}{I_S}\right) - (1 + K)^{\theta}\right]\)$

où :

  • \(N\) = Notionnel

  • \(I_T, I_S\) = Indices d’inflation aux dates T et S

  • \(K\) = Taux fixe

  • \(\theta\) = Facteur d’accroissement pour la période [S,T]

📈 Inflation Year-on-Year Swaps (InflSwapYOY)#

Description : Swaps échangeant un taux fixe contre le taux d’inflation annuel.

Formule de Payoff : $\(\text{Payoff} = N \times \left[\frac{I_T}{I_S} - (1 + K)\right]\)$

💰 Inflation Asset Swaps (InflAssetSwap)#

Description : Asset swaps avec composante d’inflation intégrée.

🏦 Interest Rate Real Rate Swaps (IRSwapRealRate)#

Description : Swaps de taux d’intérêt réels ajustés pour l’inflation.

📊 Single Inflation Index (IRIndexSingle_INFLATION)#

Description : Produits basés sur un indice d’inflation unique.

🏗️ Model Description#

Framework Théorique#

Taux d’Intérêt Déterministes#

Dans le framework DBOPT, les taux d’intérêt dans chaque devise sont supposés être une fonction déterministe du temps.

Représentation Mathématique : $\(r(t) = f_{\text{deterministic}}(t)\)$

Cette approche simplifie considérablement les calculs tout en maintenant la précision pour les produits sans optionalité.

Inflation Déterministe#

L’inflation suit également un modèle déterministe où :

\[I(t) = I_0 \times \exp\left(\int_0^t i(s) ds\right)\]

où \(i(s)\) est le taux d’inflation instantané déterministe.

Implémentation Technique dans Our project#

Construction de Courbe d’Inflation#

// Implémentation de la courbe d'inflation avec ajustements saisonniers
class inflation_curve : public market_data {
private:
    datetime base_date_;
    ptr<interpolator<std::vector<double>, datetime, double>> inverse_dfs_;
    std::vector<double> seasonality_adjustment_;
    ptr_const<inflation_fixing> historic_fixings_;
    double cpi_b_;

public:
    inflation_curve(
        datetime valuation_date,
        datetime base_date,
        interpolation_enum interpolation_type,
        std::vector<datetime> node_dates,
        std::vector<double> rates,
        std::vector<double> seasonality_adjustment,
        const ptr_const<inflation_fixing>& historic_fixings);
    
    double cpi(const datetime& to, const tenor& observation_lag) const;
    void initialize();
};

Cette classe C++ dans Our project implémente la construction complète de courbes d’inflation avec support pour les ajustements saisonniers et les fixings historiques.

Calcul CPI avec Ajustements#

double inflation_curve::cpi(const datetime& to, const tenor& observation_lag) const {
    const auto date = datetime_helper::add_tenor(to, observation_lag, -1);
    
    // Utilisation des fixings historiques pour les dates passées
    if (date < base_date_) {
        return historic_fixings_->cpi(date);
    }
    
    // Application des ajustements saisonniers si disponibles
    if (XSIGMA_LIKELY(!seasonality_adjustment_.empty())) {
        year_month_day mdy(date);
        return cpi_b_ * inverse_dfs_->interpolate(date) *
               seasonality_adjustment_.at(year_month_day(date).month - 1);
    } else {
        return cpi_b_ * inverse_dfs_->interpolate(date);
    }
}

Cette méthode C++ montre comment Our project calcule les valeurs CPI en tenant compte des ajustements saisonniers et des données historiques.

Méthodologie de Construction de Courbe#

Instruments de Marché#

DBA permet la création de courbes d’inflation à partir d’un ensemble d’instruments :

  • Inflation forwards

  • Zero-coupon swaps

  • Year-on-year swaps

Facteurs d’Actualisation#

La construction produit un continuum de facteurs d’actualisation théoriques \(D_I(t)\) avec maturité \(t\) qui reprice ces instruments.

Configuration DBA :

  • Type d’objet : “CURVE”

  • UseType : “INFLATION”

  • CurveType : “SPREAD”

Relation avec les Indices d’Inflation#

Étant donné les facteurs d’actualisation, les valeurs d’inflation pour les dates après la dernière date CPI sont données par :

\[I(\text{date}) = \frac{I_{\text{LastCPIDate}}}{D^I(\text{date} + \text{InflationDelayOutput})}\]

où : $\(\text{InflationDelayOutput} = \text{ValuationDate} - \text{LastCPIDate}\)$

Bootstrapping et Interpolation#

Méthodes de Construction#

La courbe \(D^I(t)\) est bootstrappée à partir d’instruments input dans une table “Swap” avec :

  • SettlementDate (optionnel, override la SettlementDate de la courbe)

  • Maturity (maturité de l’instrument)

  • ZeroRate (taux de coupon du marché)

Équations de Bootstrapping#

Pour les ZCSwaps : $\(\text{ZeroRate} = \frac{1}{\theta} \left(\frac{D^I(S)}{D^I(T)} - 1\right)\)$

Pour les YoY swaps : $\(\text{ZeroRate} = \frac{D^I(S)}{D^I(T)} - 1\)$

où \(S\) est la date de début et \(T\) la maturité.

🔧 Feeder and Servicer Models#

Architecture des Modèles#

Les modèles DBOPT fonctionnent dans un écosystème de modèles interconnectés :

Feeder Models#

  • Market Data Feeds : Courbes de taux et indices d’inflation

  • Calibration Models : Paramètres de construction de courbe

  • Historical Data : Fixings CPI historiques

Servicer Models#

  • Risk Calculation Engines : VaR, sensibilités

  • Stress Testing Frameworks : CCAR, IRRBB

  • Reporting Systems : EOD valuations

Méthodologie de Validation#

🔍 Phase 1 : Validation Théorique#

  • Vérification des fondements mathématiques

  • Validation des hypothèses du modèle

  • Analyse de cohérence théorique

🔍 Phase 2 : Tests d’Implémentation#

  • Tests de réplication de prix

  • Validation des algorithmes de construction de courbe

  • Vérification de la précision numérique

🔍 Phase 3 : Tests de Stress#

  • Stabilité sous stress de taux

  • Comportement sous chocs d’inflation

  • Tests de sensibilité paramétrique

🔍 Phase 4 : Validation Réglementaire#

  • Conformité CCAR

  • Tests IRRBB

  • Validation multivariate VaR

Critères d’Acceptation#

  • Précision : Erreurs < 0.01 bp pour la réplication

  • Stabilité : Convergence sous tous les scénarios de stress

  • Performance : Temps de calcul < seuils définis

  • Robustesse : Comportement prévisible sous conditions extrêmes
