Executive Summary#

📊 Global Model Risk Assessment#

Le Global Model Risk Assessment pour les modèles DBOPT-RateDeterministic_InflationDeterministic et DBX-RateDeterministic_InflationDeterministic présente les évaluations suivantes :

Critère

Rating

Justification

Complexité

LOW(1)/LOW(2)

Basé sur le jugement SME du validateur

Incertitude

LOW(1)/LOW(2)

Les méthodologies du modèle ne présentent pas de faiblesses ou limitations significatives conduisant à un faible niveau d’incertitude

Dépendance

HIGH(1)/HIGH(2)

Les modèles de pricing ont une dépendance élevée

Table 2: Évaluation des risques du modèle pour les deux modèles dans le scope de ce rapport: (1) DBOPT-RateDeterministic_InflationDeterministic, (2) DBX-RateDeterministic_InflationDeterministic.

🎯 Méthodologie d’Évaluation#

Rating d’Incertitude#

Le rating d’incertitude a été déterminé en considérant :

  • Les inputs du modèle

  • Les limitations pertinentes du modèle

  • La représentation des facteurs de risque requise pour les produits modélisés

  • La nature de l’implémentation numérique

  • Les approches de calibration employées

Rating de Complexité#

Le rating de complexité a été déterminé en considérant :

  • Le nombre et la nature des facteurs de risque modélisés, incluant leur interdépendance

  • L’approche de calibration employée

  • La spécification mathématique du modèle

  • La sophistication de l’environnement de traitement plus large dont le modèle fait partie

Dépendance des Modèles#

La dépendance de tous les modèles de pricing est élevée car les outputs du modèle définissent les métriques officiellement reportées plutôt que de simplement les informer. La réduction de la dépendance aux outputs des modèles de pricing est inappropriée.

✅ Résumé des Principaux Résultats de Validation#

Objectifs des Modèles#

Les modèles DBOPT-RateDeterministic_InflationDeterministic et DBX-RateDeterministic_InflationDeterministic sont des modèles de pricing utilisés pour les objectifs suivants :

📈 Évaluation et Reporting#

  • Génération de mesures de fair value d’actifs et passifs reportées dans les livres et registres officiels

  • Génération de sensibilités de risque pour la gestion des risques End of Day

  • Alimentation des calculs de Risk and Capital

🔄 Tests de Stress#

  • Re-pricing des positions sous scénarios de stress comme modèle servicer

  • Tests de stress réglementaires (CCAR)

  • Calculs Interest Risk in Banking Book (IRRBB) Delta Economic Value of Equity (EVE)

  • Full revaluation de (Stress) Value at Risk (SVaR/HistSim/Multivariate)

Développement et Maintenance#

Le modèle est développé et maintenu par DBAnalytics et fourni via :

  • DLLs pour utilisation dans Microsoft Excel

  • Python et Perl

  • Disponible dans Kannon et Risk Engine

🏗️ Scope des Produits#

Caractéristiques des Produits#

Le modèle est utilisé pour pricer des produits dépendant des taux d’intérêt et de l’inflation qui ne présentent aucune optionalité, ni sur les taux d’intérêt ni sur l’indice d’inflation.

Propriétés Linéaires#

  • Tous les produits sont linéaires par rapport aux courbes de taux d’intérêt et d’inflation

  • La courbe de taux d’intérêt intervient essentiellement dans la construction des facteurs d’actualisation

  • La courbe d’inflation \(I(t)\) est utilisée pour exprimer la valeur d’inflation à la date \(T\)

Framework Théorique#

// Implémentation de la courbe d'inflation dans Our project
double inflation_curve::cpi(const datetime& to, const tenor& observation_lag) const {
    const auto date = datetime_helper::add_tenor(to, observation_lag, -1);
    
    if (date < base_date_) {
        return historic_fixings_->cpi(date);
    }
    
    if (XSIGMA_LIKELY(!seasonality_adjustment_.empty())) {
        year_month_day mdy(date);
        return cpi_b_ * inverse_dfs_->interpolate(date) *
               seasonality_adjustment_.at(year_month_day(date).month - 1);
    } else {
        return cpi_b_ * inverse_dfs_->interpolate(date);
    }
}

Cette implémentation C++ dans Our project montre comment la courbe d’inflation calcule les valeurs CPI avec ajustements saisonniers et fixings historiques.

🔧 Approche Déterministe#

Principe Fondamental#

L’hypothèse ‘déterministe’ ne signifie pas que les payoffs des produits sous ce modèle sont supposés être déterministes. Elle signifie simplement que, en raison de l’absence d’optionalité, leur PV est indépendant de la volatilité.

Implications Pratiques#

  • Le pricing des produits est indépendant de toute modélisation particulière des taux d’intérêt ou de la courbe d’inflation

  • Basé uniquement sur des arguments généraux de non-arbitrage

  • Le framework DBOPT est une méthodologie DBA pour pricer des titres ayant une solution fermée

🧪 Validation Testing Principal#

Tests de Validation Effectués#

La validation principale effectuée dans la Section 4 vérifie le comportement des modèles pour le pricing de produits d’inflation avec un payoff linéaire :

Résultats de Validation#

Tous les tests suggèrent que le modèle est adapté à son usage. Le modèle a été revu pour DBAnalytics v1300.

Modèles Recommandés#

DBOPT-RateDeterministic_InflationDeterministic#

Configuration : DBOPT-RatesMarketDeterministic_InflationDeterministic

Produits approuvés :

  • InflSwapZC* - Inflation Zero Coupon Swaps

  • InflSwapYOY* - Inflation Year-on-Year Swaps

  • InflAssetSwap* - Inflation Asset Swaps

  • IRSwapRealRate* - Interest Rate Real Rate Swaps

DBX-RateDeterministic_InflationDeterministic#

Configuration : DBX-ShortRateDeterministic_InflationDeterministic

Produits approuvés :

  • IRIndexSingle_INFLATION*

Les produits marqués avec “” indiquent que le modèle approuvé est primaire.*
